# Домашнее задание к занятию "6.1. Типы и структура СУБД"

## Задача 1

- Электронные чеки в json виде - Документо-ориентированная. Сетевая - иерархическая с множеством узлов
- Склады и автомобильные дороги для логистической компании.
- Генеалогические деревья. - Иерархические. В иерархической модели данные организованы в древовидную структуру
- Кэш идентификаторов клиентов с ограниченным временем жизни для движка аутенфикации. - Ключ-значение
- Отношения клиент-покупка для интернет-магазина. - Реляционные. Имеют четкую структуру и зависимости

## Задача 2
- Данные записываются на все узлы с задержкой до часа (асинхронная запись): CA, EL-PC
- При сетевых сбоях, система может разделиться на 2 раздельных кластера: AP, PA-EL
- Система может не прислать корректный ответ или сбросить соединение: CP, PA-EC

## Задача 3
Согласно лекции эти принципы противоречат друг другу. По ACID - данные согласованные, а по BASE - могут быть неверные.
